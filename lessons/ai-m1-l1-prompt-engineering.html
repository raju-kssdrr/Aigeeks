Prompt Engineering & Tuning

Core Concepts

System Prompts

A system prompt sets the persona, constraints, and context for the entire conversation. It is the highest-priority instruction in the context window. Well-crafted system prompts reduce hallucination, enforce output format, and define the model's role precisely.

Few-Shot Prompting

Provide 2–5 input/output examples directly in the prompt. The model infers the pattern and applies it to new inputs. Especially effective for classification, formatting, and structured extraction tasks where zero-shot fails.

Chain-of-Thought (CoT)

Appending "Let's think step by step" or providing step-by-step examples causes the model to reason through intermediate steps before giving a final answer. Dramatically improves accuracy on multi-step math, logic, and planning tasks.

Temperature & Sampling

Temperature (0–2) controls randomness. Low temperature (0–0.3) = deterministic, good for code and facts. High temperature (0.7–1.2) = creative, good for brainstorming. Top-p (nucleus sampling) limits token selection to the top cumulative probability mass.

# System prompt pattern
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    system="You are a senior Python engineer. Respond with clean, production-ready code only. No explanations unless asked.",
    messages=[
        {"role": "user", "content": "Write a function to parse ISO 8601 dates."}
    ]
)
print(response.content[0].text)
