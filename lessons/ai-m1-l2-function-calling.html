Function Calling & Tool Use

⏱ 45 min 📚 AI Engineering for Devs 🧩 Lesson 1.2
Tool DefinitionsParallel CallsError HandlingStructured Outputs
Core Concepts

Tool Definitions

Functions are exposed to the model as JSON schemas describing the name, description, and parameters. The model decides when to call a tool based on the conversation context. A clear, specific description is the single most important factor in whether the model calls the right tool.

Parallel Tool Calls

Modern LLM APIs support parallel tool calls — the model emits multiple tool_use blocks in a single response. This allows concurrent API calls (e.g. fetch weather + fetch calendar simultaneously) without sequential round-trips, cutting latency significantly.

Error Handling

Always return a tool_result even on failure. Pass the error message as the result content so the model can reason about it and retry with corrected parameters. Never leave a tool call without a corresponding result or the context will be malformed.

Structured Outputs

Use tool calling as a structured output mechanism even when you don't need external execution. Define a schema for the desired output shape and the model will fill it reliably — far more consistent than parsing free-text JSON from the response.

import anthropic
import json

client = anthropic.Anthropic()

tools = [
    {
        "name": "get_weather",
        "description": "Get current weather for a city.",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "City name"},
                "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
            },
            "required": ["city"]
        }
    }
]

def get_weather(city, unit="celsius"):
    # Real implementation would call a weather API
    return {"city": city, "temp": 22, "unit": unit, "condition": "sunny"}

messages = [{"role": "user", "content": "What's the weather in Tokyo?"}]

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

# Handle tool call
if response.stop_reason == "tool_use":
    tool_use = next(b for b in response.content if b.type == "tool_use")
    result = get_weather(**tool_use.input)
    
    messages += [
        {"role": "assistant", "content": response.content},
        {"role": "user", "content": [{"type": "tool_result", "tool_use_id": tool_use.id, "content": json.dumps(result)}]}
    ]
    
    final = client.messages.create(model="claude-opus-4-7", max_tokens=1024, tools=tools, messages=messages)
    print(final.content[0].text)
