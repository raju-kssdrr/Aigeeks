Module 3: Agentic Systems

LangChain vs LangGraph

⏱ 60 min 📚 AI Engineering for Devs 🧩 Lesson 3.1
Chain TypesGraph StateCheckpointingHuman-in-the-Loop
Core Concepts

LangChain Chains

LangChain chains are linear pipelines: input → LLM → output → next step. Good for simple, predictable flows. LCEL (LangChain Expression Language) uses the pipe operator (|) to compose runnables. Easy to prototype but becomes unwieldy for branching, looping, or stateful workflows.

LangGraph State Machines

LangGraph models agent workflows as directed graphs where nodes are functions and edges are transitions. State is a typed dict passed between nodes. Enables loops, conditionals, and parallel branches — things chains cannot do. The go-to for production agentic systems.

Checkpointing & Memory

LangGraph supports persistent checkpointers (SQLite, PostgreSQL) that save graph state after every node. Enables pause/resume, multi-turn conversations with full history, and recovery from failures mid-workflow without restarting from scratch.

Human-in-the-Loop

Interrupt the graph at any node to request human approval or input. The graph pauses, state is persisted, and execution resumes when the human responds. Critical for high-stakes actions (sending emails, deploying code, making purchases) where autonomous execution is unacceptable.

from langgraph.graph import StateGraph, END
from langchain_anthropic import ChatAnthropic
from typing import TypedDict, Annotated
import operator

llm = ChatAnthropic(model="claude-opus-4-7")

class AgentState(TypedDict):
    messages: Annotated[list, operator.add]
    next_action: str

def call_llm(state: AgentState):
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

def should_continue(state: AgentState):
    last = state["messages"][-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"
    return END

def run_tools(state: AgentState):
    # Execute tool calls from last message
    results = []
    for tc in state["messages"][-1].tool_calls:
        result = execute_tool(tc["name"], tc["args"])  # your tool executor
        results.append({"role": "tool", "content": str(result), "tool_call_id": tc["id"]})
    return {"messages": results}

# Build graph
graph = StateGraph(AgentState)
graph.add_node("llm", call_llm)
graph.add_node("tools", run_tools)
graph.set_entry_point("llm")
graph.add_conditional_edges("llm", should_continue)
graph.add_edge("tools", "llm")

agent = graph.compile()
result = agent.invoke({"messages": [{"role": "user", "content": "Search for the latest AI papers"}]})
