RLHF & DPO

Core Concepts

RLHF Overview

Reinforcement Learning from Human Feedback: (1) SFT — fine-tune on demonstrations; (2) Reward Model — train a classifier to predict human preferences from paired comparisons; (3) PPO — use RL to optimize the policy against the reward model while staying close to the SFT baseline via KL divergence penalty.

DPO (Direct Preference Optimization)

DPO skips the reward model entirely. It directly fine-tunes the LLM on preference pairs (chosen vs. rejected) using a closed-form loss derived from the RLHF objective. Simpler to implement, more stable than PPO, and achieves comparable results. The go-to alignment method for practitioners.

Preference Dataset Format

DPO requires triplets: (prompt, chosen_response, rejected_response). Chosen = the preferred/correct/safer output. Rejected = the worse output. Sources: human annotators, AI labeling with a strong model, or synthetic generation with GPT-4. Quality of the rejected example matters — too easy = poor signal.

Practical DPO Tips

Learning rate: 1e-6 to 5e-6 (much lower than SFT). Beta (KL penalty) = 0.1 is a safe default. Train on top of a SFT checkpoint, not a raw base model. Monitor reference model divergence — if the policy drifts too far, outputs degrade. 500–5000 preference pairs is usually sufficient for targeted alignment.

# DPO training with TRL
from trl import DPOTrainer, DPOConfig
from transformers import AutoModelForCausalLM, AutoTokenizer
from datasets import load_dataset

model_name = "meta-llama/Llama-3.2-3B-Instruct"
model = AutoModelForCausalLM.from_pretrained(model_name)
ref_model = AutoModelForCausalLM.from_pretrained(model_name)  # frozen reference
tokenizer = AutoTokenizer.from_pretrained(model_name)

# Dataset must have: prompt, chosen, rejected columns
dataset = load_dataset("json", data_files="data/preferences.jsonl")["train"]
# Example row: {"prompt": "Explain quantum...", "chosen": "Great answer", "rejected": "Bad answer"}

trainer = DPOTrainer(
    model=model,
    ref_model=ref_model,
    args=DPOConfig(
        output_dir="./dpo-output",
        num_train_epochs=1,
        per_device_train_batch_size=2,
        gradient_accumulation_steps=4,
        learning_rate=5e-7,
        beta=0.1,          # KL penalty coefficient
        bf16=True,
        logging_steps=10,
    ),
    train_dataset=dataset,
    tokenizer=tokenizer,
)
trainer.train()
